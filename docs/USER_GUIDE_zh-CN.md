# PsyReaSFX 用户使用说明书

**适用版本：** PsyReaSFX 0.7.22 Beta 26  
**作者：** Psysia  
**宿主：** REAPER 7.x

本手册按照实际工作任务介绍产品。逐版本的新增、修复和兼容性变化请阅读独立的[更新日志](CHANGELOG_zh-CN.md)。

## 1. 欢迎使用 PsyReaSFX

PsyReaSFX 是运行在 REAPER 内部的音效资产工作区，把素材库管理、列表波形、元数据搜索、试听、整理、时间线放置和处理后导出集中在一个可停靠界面中。

软件使用两个相互关联的存储概念：

- **逻辑音效库**：左侧导航中显示的名称和分组。
- **实体来源文件夹**：硬盘上的真实路径。

一个逻辑音效库可以包含多个实体来源。因此，同一个制作库可以分布在不同硬盘或位置，而不必破坏人们理解素材所依赖的分组结构。

PsyReaSFX 默认采用非破坏性工作方式。收藏、标记、工作流状态、集合、保存搜索、Region 与元数据编辑保存在本地数据库。只有明确执行 Transfer 时，才会在目标位置生成新的音频文件。

## 2. 环境要求与安装

### 必需

- REAPER 7.x
- ReaImGui 0.10 或更高版本

### 建议安装

- SWS Extension

SWS 用于波形点击精确定位、选区试听、高级 Preview 参数、声道监听，以及拖到 REAPER 编排区时的时间和轨道识别。没有 SWS 时，数据库、搜索和基础浏览仍可使用，但高级试听与拖放能力会受限。

### 使用 ReaPack 安装

1. 在 REAPER 中打开 `扩展 → ReaPack → 导入仓库…`。
2. 添加：

   ```text
   https://github.com/Psysia/PsyReaSFX/raw/main/index.xml
   ```

3. 同步软件包。
4. 搜索并安装 `PsyReaSFX`。
5. 打开 REAPER 动作列表，运行 PsyReaSFX；需要时给它绑定快捷键。

ReaPack 会安装主脚本、应用图标和 Orbitron 品牌字体，以后也通过同一个仓库更新。

### 手动安装

请保持发布包结构完整：

```text
PsyReaSFX_vX_X_X/
├─ PsyReaSFX_vX_X_X.lua
└─ assets/
   ├─ brand/
   └─ fonts/
```

在动作列表中加载 Lua 文件。即使品牌字体缺失，界面也会安全回退到 REAPER 常规字体。

## 3. 第一次启动

1. 如果左侧导航已隐藏，按 `F9` 打开。
2. 点击**新建音效库**。
3. 输入名称。此时只创建逻辑音效库，不要求选择路径。
4. 添加一个或多个实体来源，或从 Windows 资源管理器把文件夹拖到新库上。
5. 等待导入任务完成。
6. 在结果波形中点击，即可从对应位置开始试听。

首次访问某个文件夹时，软件需要建立元数据和波形缓存；再次访问会复用磁盘缓存。

## 4. 工作区导览

### 顶部工具栏

导航、元数据和专注模式开关统一放在品牌标记右侧，形成一个清晰的工作区控制组。搜索框处于中央，旁边提供自动试听、扫描、帮助和设置等常用操作。

工具栏图标在静止时不显示外框；悬停时才显示点击范围和强调色，已经启用的状态则持续可见。将鼠标停留片刻可显示本地化功能说明。

左上角 PsyReaSFX 标记由程序直接绘制为矢量图形，因此在紧凑尺寸和高 DPI 下仍然清晰。产品字标会优先使用随包附带的 Orbitron 字体。

### 左侧导航

导航包括：

- 全部素材、收藏、最近插入和试听历史；
- 逻辑音效库及其实体来源；
- 播放列表和项目素材箱；
- 保存搜索；
- 工作流状态筛选。

点击库名前的箭头展开或折叠实体来源。点击逻辑库可查看所有来源的集合；点击某个来源只筛选该路径。

**素材、音效库、集合、保存搜索、工作流、活动**六个分组都可以点击标题独立折叠。PsyReaSFX 会保存每个分组的展开状态，因此无需隐藏整个左栏，也能长期保持紧凑的导航布局。

### 结果列表

中央列表由固定表头和虚拟化结果行组成。波形、文件名、描述、状态、分类、时长、格式、Artwork、音效库和路径等字段可以分别显示或隐藏。素材没有封面时，Artwork 单元格会直接留空，不再重复显示占位图。

### 元数据检查器

右侧面板显示 Artwork、文件属性和数据库元数据。Artwork 可以固定在顶部，下面的元数据独立滚动。

### 预览工作区

底部区域包括大波形、选区和播放位置、Region、响度、试听参数和 REAPER 交付操作。上下拖动中央分隔线可以调整列表与预览所占空间。

### 面板快捷键

| 快捷键 | 功能 |
|---|---|
| `F9` | 显示或隐藏左侧导航 |
| `F10` | 显示或隐藏右侧元数据 |
| `F11` | 开启或退出专注模式 |

专注模式会同时隐藏左右面板，但不会丢失它们原来的保存状态。

## 5. 音效库与实体来源

### 先建立空的逻辑音效库

点击**新建音效库**，输入名称并确认。无需马上指定文件夹；可以等硬盘或素材准备好之后再逐步加入来源。

### 添加实体来源

- 右键逻辑库，选择**添加来源文件夹…**。
- 从资源管理器把一个或多个文件夹拖到逻辑库上。
- 拖到**全部音效库**，自动建立新的逻辑父库。
- 拖到中央投放区，加入当前正在查看的逻辑库；没有适合的上下文时，软件会提供新建逻辑库的选项。

为避免重复索引，完全重复的根目录以及相互包含的父子根目录会被阻止。如果某个来源已经属于另一个库，可以转移它的逻辑归属。以上操作都不会移动或删除硬盘文件。

离线来源会继续显示，因此移动硬盘、网络盘或盘符变化不会让库关系直接丢失。

### 浏览索引文件夹层级

主工具栏下方有常驻的 **Path 路径栏**。点击后会在主窗口内部展开目录
浏览器，层级依次为：

1. 全部逻辑音效库；
2. 每个逻辑库拥有的实体来源；
3. 直接或间接包含已索引音频的真实子目录。

单击箭头只展开或折叠分支；单击名称会关闭浏览器，并显示该目录及全部
下级目录中的音频。Path 栏右侧的 `×` 只清除目录条件，同时保留所属
逻辑库筛选。

目录展开状态和当前路径会跨启动保存。目录树从 `index_v3.tsv` 分批建立，
浏览时不会触发硬盘扫描。实际文件夹结构变化后，再运行普通音效库扫描
即可更新。

### Artwork 归属

每个实体来源独立拥有自动找到或手动指定的封面。同一逻辑库中，一个来源的封面不会套用到其他来源。

自动识别支持封面与音频放在一起、放在来源子目录，以及
`1. Audio / 2. Artwork` 这类带编号的兄弟目录结构。识别目录角色前会
去除排序编号，兼容 Artwork、Cover、Images、Graphics、Thumbnail
等名称。嵌套搜索具有严格的深度与目录数量限制，适合大型商业素材库。

同一位置存在多个候选图片时，PsyReaSFX 会优先选择正方形或接近 1:1 的图片，再比较常见封面文件名、图片分辨率和稳定的文件名顺序。PNG、JPEG、BMP、TGA 只读取文件头尺寸，不会在筛选阶段完整解码图片，避免大型库导入时增加明显负担。

右键展开后的实体来源，可以指定、重新发现或清除 Artwork。元数据面板中为某个单独素材指定的封面优先级最高。

## 6. 搜索与筛选

### 普通文字

普通搜索会联合匹配文件名、路径、描述、关键词、Category、SubCategory、CatID、库名和 UCS 派生字段。

```text
cinematic whoosh
metal impact
magic ui
```

### 字段筛选

```text
category:impact
subcategory:metal
catid:WPN
library:boom
description:heavy
keywords:magic
channels:2
status:candidate
marked:true
played:true
```

字段和普通文字可以组合。词语前加 `-` 表示排除：

```text
whoosh category:movement -long
```

### 保存搜索

保存搜索可以记录查询文字、库、集合、工作流筛选和排序方向。适合建立重复使用的审核视图，不需要为了不同查看方式复制素材文件。

## 7. 结果分栏

右键固定表头可以选择显示字段。拖动分隔线改变列宽；双击分隔线恢复默认宽度。

时长采用固定 `MM:SS.mmm` 格式，例如 `00:04.947`。

字段总宽度超过工作区时，把鼠标放在表头或结果行上，使用 `Shift + 鼠标滚轮` 横向查看。表头和内容保持同步，最右侧字段始终可以到达，同时界面不常驻一条横向滚动条。

隐藏字段不会参与绘制，因此精简分栏也能降低大型结果集的界面负担。

### Artwork 列

Artwork 会检查实体来源根目录、直接封面子目录，以及产品父目录下
符合封面角色的兄弟目录。它同时兼容封面和 WAV 同目录，以及
`1. Motion Graphics / 2. Artwork` 这样的结构。优先文件名包括
`artwork`、`cover`、`folder`、`front`、`album` 和 `thumbnail`，
支持 PNG、JPG、JPEG。

发现任务采用低优先级队列，同时缓存“该文件夹没有封面”的结果。隐藏 Artwork 列后，不再请求列表缩略图。

## 8. 选择和整理素材

| 操作 | 结果 |
|---|---|
| 单击 | 单选 |
| `Ctrl`+单击 | 添加或取消一个素材 |
| `Shift`+单击 | 连续范围选择 |
| `Ctrl+A` | 全选当前结果 |
| `F` | 收藏或取消收藏 |
| `M` | 标记或取消标记 |

收藏适合长期选择；标记适合临时审核。未标记、候选、已采用、已排除属于另一套独立的工作流状态。

播放列表用于可复用的素材集合，项目素材箱用于项目候选。它们都是虚拟集合，不会复制源文件。

### 已播放颜色

素材实际开始试听后，文字可以变为自定义的已播放颜色。可以从工具栏或设置清除本次高亮，也可以在误关窗口后恢复上次会话。本地试听历史与这项视觉状态相互独立。

波形状态显示优先级为：

```text
选中 > 已标记 > 已播放（启用时）> 普通
```

## 9. 列表波形试听

- 点击缩略波形任意位置，选择素材并从该时间开始试听。
- 当前正在试听的行显示迷你播放指针。
- `Space` 播放或停止。播放统一经过源边界安全淡入；停止使用 SWS 在音频线程中执行的采样级淡出，不再使用按界面帧更新的音量动画。
- `上/下方向键` 切换结果。
- 开启自动试听后，切换选择会自动开始播放。

列表波形默认 256 点，也可提高到 512 点。整库高精度预缓存支持 2048 或 4096 点，并将结果写入磁盘，不把整个素材库同时放进内存。

## 10. 大波形预览

### 浏览

| 鼠标操作 | 功能 |
|---|---|
| 滚轮 | 以鼠标位置为中心缩放 |
| `Shift`+滚轮 | 水平平移 |
| 中键拖动 | 水平平移 |
| 双击 | 重置缩放 |
| 右键拖动 | 擦播 |

### 建立并使用选区

在波形上左键拖动建立选区。之后可以只试听选区、自动循环、插入、导出，或直接按住选区拖拽区域放到 REAPER 编排区。

### 分声道显示

开启 `设置 → 波形 → 显示独立声道` 后，大波形会显示：

- 单声道：`M`
- 立体声：`L`、`R`
- 多声道：`CH 1`–`CH 8`

标尺、选区、Region 和播放指针跨越全部声道。分声道大波形使用 RWF3 缓存，已有的列表缩略波形仍可继续使用。

声道监听图标会根据源文件自动变化：

- 单声道和立体声文件：左键在立体声/原始、左、右、单声道之间循环；图标中的竖线会移动到左侧、右侧或中间，直接表示当前模式。右键恢复立体声/原始模式。
- 三声道以上文件：左键在主预览控制条内部展开或收起声道条，不会创建新窗口，因此试听和应用快捷键始终可用。
- 内联声道条中，单击只选一个声道，`Ctrl`+单击追加或取消，`Shift`+单击连续选择；鼠标位于声道条上时按 `Ctrl+A` 全选。右键单个声道可立即聚焦它的波形；右键多声道图标恢复全部声道。
- 已选声道保持完整波形亮度；未选声道降低亮度但不消失，方便保留多声道上下文。
- 只有全部声道确实处于选中状态时，“全部声道”才显示选中底色。

SWS 预览接口没有公开任意多声道源声道隔离能力。因此，PsyReaSFX 对单声道/立体声模式执行准确的居中监听；三至八声道的选择和右键聚焦只作用于波形显示，实际声音仍遵循 REAPER 的多声道设备路由，状态栏也会明确提示。这样不会为了试听偷偷建立临时轨道或修改工程。

### 响度显示

预览标题可以显示 LUFS-I、最大 LUFS-M、最大 LUFS-S 和 True Peak。分析按需执行并缓存。估算响度匹配只用于快速比较，不应视为符合标准的交付响度测量。

## 11. Region 与瞬态建议

Region 可以保存长录音中的有效范围，不修改源文件。手动 Region 标记为 `[M]`；自动瞬态建议标记为 `[T]`。

瞬态检测可以调整：

- dBFS 阈值；
- 包络平滑；
- 最小间隔；
- 前置和后置时间；
- 最大建议数量；
- 替换或追加行为。

可以撤销上次检测、清除全部瞬态建议，或在 Region 列表中单独删除一个区间。

## 12. 试听控制

| 参数 | 范围或模式 |
|---|---|
| Pitch | -24 到 +24 半音 |
| Rate | 0.25x 到 4x |
| Gain | -36 dB 到 +18 dB |
| 音高行为 | 保持音高或变速变调 |
| 方向 | 正向或反向 |
| 循环 | 完整文件或当前选区 |
| 声道 | 立体声/原始、左、右、单声道；CH 1–8 波形聚焦 |

常用 Pitch 和 Rate 可通过预设快速选择。这些参数会影响试听；在 Transfer 中启用时也会进入处理结果。

拖动参数卡或使用鼠标滚轮可以连续调整。双击右侧数值会在原位置进入精确数字输入，输入时用强调色高亮，不改变卡片尺寸或排版；按 `Enter` 或点击外部确认，按 `Esc` 取消。参数编辑会独占当帧键盘输入，因此确认数值不会再触发全局“插入素材”快捷键。双击标签或滑轨可恢复默认值。

## 13. 放入 REAPER

PsyReaSFX 可以：

- 插入当前轨道；
- 插入新轨道；
- 按 BWF 时间戳放置；
- 只插入当前选区；
- 将多个已选文件纵向分轨放置；
- 把列表素材或波形选区拖入编排区。

这里的拖放是 PsyReaSFX 到 REAPER 的内部操作，不是 Windows 通用文件拖放。建议安装 SWS，以准确识别目标时间和轨道。

## 14. Transfer：生成处理后的文件

从底部工具栏打开 Transfer，或按 `Ctrl+T`。Transfer 会生成新文件；除非主动选择覆盖策略并确认，否则不会覆盖源素材。

<p align="center">
  <img src="../assets/screenshots/transfer-settings-zh.png" alt="PsyReaSFX 中文 Transfer 设置" width="820">
</p>

### 输出与命名

选择输出文件夹，并使用以下变量建立命名模板：

| 变量 | 内容 |
|---|---|
| `{name}` | 不含扩展名的源文件名 |
| `{category}` | Category 元数据 |
| `{subcategory}` | SubCategory 元数据 |
| `{library}` | 逻辑音效库名 |
| `{index}` | 两位批次序号 |
| `{date}` | `YYYYMMDD` 导出日期 |
| `{region}` | Region 名、`selection` 或 `full` |
| `{pitch}` | 变体 Pitch，单位为半音 |
| `{rate}` | 变体播放速率 |
| `{gain}` | 变体 Gain，单位为 dB |
| `{direction}` | `normal` 或 `reverse` |
| `{variant}` | 完整的安全变体标签 |
| `{variant_index}` | 两位变体序号 |

模板展开后会处理 Windows 非法文件名字符，并可选择统一转为小写。

### 当前支持范围

Transfer 的互斥选项采用分段选择器。当前导出范围、格式、采样率、声道、
标准化和重名策略会使用主题强调色、高对比文字和独立边框显示。

- 范围：当前完整素材、当前选区或全部已选素材。
- 格式：明确的 WAV 16-bit、24-bit、32-bit PCM，或 REAPER 默认 FLAC 编码器。
- 采样率：跟随源文件、44.1、48、96、192 kHz。
- 声道：跟随源文件、单声道、立体声。
- 处理：当前 Pitch、Rate、Gain、Reverse 和 Preserve Pitch。
- 收尾：Fade，以及 Peak、True Peak、RMS-I 或 LUFS-I 标准化。
- 交付完整性：尽可能保留目标格式支持的源元数据；WAV 16-bit 可启用
  抖动和噪声整形。
- 源文件智能尾音：对波形选区按 dBFS 阈值查找最后一个有效声音，
  并受最大时长与安全留白限制。
- 重名：自动编号、跳过，或明确确认后覆盖。
- 完成：可把输出文件自动插回 REAPER；任务正常结束且至少成功输出一个
  文件时，也可自动打开输出目录。

### 批量变体

开启**批量变体**后，可以分别输入 Pitch、Rate、Gain 数值列表；逗号、
空格和分号都可作为分隔符。留空表示沿用主界面当前值。PsyReaSFX 会
生成这些列表的笛卡尔组合，并可选择同时加入正向和反向两种方向。

每个参数最多接受 16 个不重复数值，单个素材最多生成 128 个变体，
单次任务最多 4096 项。当命名模板没有任何变体字段时，默认的自动
安全后缀会追加类似 `p0_r1_g0_fwd` 的标签。除非模板本身已经保证
输出名唯一，否则建议保持该选项开启。

### 大型任务、停止与报告

批量 Transfer 每次只调度一个渲染，在文件之间把控制权交还界面。
底部显示已完成数和总任务数。**当前文件后停止**不会在 REAPER 正在
写文件时强行中断，而是阻止下一个任务开始。

最近一次任务会在 PsyReaSFX 数据目录写入
`transfer_report_latest.tsv`，逐项记录状态、源路径、变体、输出路径、
消息和耗时；Transfer 面板可直接打开该目录。
Transfer 设置、最近输出和任务报告中的目录按钮统一使用同一个系统打开
流程。Windows 下优先使用 SWS，必要时回退到非阻塞系统命令。

覆盖模式采用安全提交顺序：先渲染临时文件，再把旧目标移动到唯一
备份，提交新文件，最后删除备份。如果提交失败，会尝试恢复原文件，
不会在渲染开始前直接删除目标。

Transfer 会临时建立干声媒体对象，使用 REAPER 的“渲染所选媒体对象”机制，然后恢复原来的渲染设置、选择、光标、时间选区和工程脏状态。无论成功或失败，临时对象都会清理。

智能尾音只分析选区之后仍存在于源文件中的音频，不会渲染工程轨道
FX、发送、文件夹路由或 Master FX 新产生的尾音。执行 Transfer 前
请停止工程播放。Reverse Transfer 需要 SWS。

## 15. 元数据与 Artwork

检查器可以编辑 Description、Keywords、Category、SubCategory、CatID、Library 和 Artwork Path。这些修改写入 PsyReaSFX 数据库，不会写回源 WAV/BWF/iXML 文件。

单选素材时，Artwork 和封面操作可以固定在上方，文件属性和元数据在下方独立滚动。可以手动选择图片、重新自动发现，或清除素材级覆盖。

当前版本不提取音频容器内部嵌入的 Artwork。

## 16. 外观与易用性

PsyReaSFX 提供默认的中性黑色“黑暗”预设，以及深藏蓝色“传统”预设。大框架底色和强调色都可以通过色盘修改；普通、选中、已播放、已标记波形，以及选区、播放指针和 Region 颜色也可独立调整。

窗口变窄时，侧边面板会临时把空间让给主工作区。工具栏点击范围保持一致，图标则仅在悬停或启用时显示背景，降低常驻边框造成的视觉噪音。

Gain 右侧的六个试听状态开关在左右面板展开时仍会保留；空间充足时保持同行，变窄后改为独立一行，不会直接消失。

设置中可以即时切换中文和英文，不需要重启。素材名称、路径和用户元数据不会被翻译。

## 17. 大型素材库与性能

PsyReaSFX 优先保证鼠标和滚动交互：

- 只绘制可见结果行；
- 隐藏字段不参与绘制；
- 文件夹树从现有索引按有界批次建立；
- 扫描、元数据、响度、Artwork 和波形任务按帧分配预算；
- 鼠标操作期间重任务主动让步；
- 当前素材和可见行优先；
- 高精度预缓存写入磁盘，不把整库保存在内存；
- “未找到封面”的结果也会缓存，避免反复扫描文件夹。

管理大型素材库时建议：

1. 初次建立时一次导入一个来源。
2. 等待当前导入任务结束后再判断数量。
3. 隐藏不使用的字段。
4. 在允许后台磁盘工作的时间执行波形预缓存。
5. 尽量把缓存放在本地高速磁盘。
6. 不要同时运行两个 PsyReaSFX 版本。

## 20. 维护与故障排查

打开 `设置 → 维护` 可以查看运行环境、复制诊断信息、修改波形缓存路径、清理缓存、重置界面、保留库路径重建数据库，或恢复出厂状态。

### 没有波形，也无法试听

- 确认源硬盘在线。
- 确认 REAPER 本身可以打开该文件。
- `._filename.wav` 是 macOS AppleDouble 旁车数据，不是音频；当前扫描会从已加载索引中过滤并移除它们，但不会删除硬盘文件。
- 源文件在索引后发生变化时，重建对应音效库。

### 第一次显示较慢

等待可见行波形任务完成，或执行高精度预缓存。之后会复用缓存。

### 点击定位或拖放位置不准

安装或更新 SWS。拖放时请在 REAPER 编排区内释放鼠标。

### 某些文字乱码

PsyReaSFX 内部统一使用 UTF-8。个别源文件元数据可能采用旧编码，可以在非破坏性元数据编辑器中覆盖该字段。

### 界面内容被裁切

放大窗口、使用专注模式、减少可见字段，或重置界面设置。报告可重复的排版问题时，请同时提供维护页复制的诊断信息和截图。
